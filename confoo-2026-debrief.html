Day 01 · Session 01 ★ Favourite
Speaker: Paul Conroy

Boosting Frontend Speed

"You don't need deep frontend expertise — just the right metrics and the right tools."
  • Measure first — Lighthouse & PageSpeed Insights; targets: CLS < 0.1, LCP < 2.5s, INP < 200ms
  • Switch images to WebP/AVIF — 25–50% smaller than JPEG with no perceptible quality loss
  • Always set explicit width & height on <img> — browser pre-allocates space, prevents CLS
  • Font subsetting + font-display: swap — cut a 200KB font to 30KB; eliminate invisible text (FOIT)
  • Reserve ad slot space in CSS upfront — avoid the 2-click penalty and protect Core Web Vitals scores

Day 01 · Session 02 ★ Favourite
Speaker: Anthony Dahanne

Java Security Essentials

"Generate and maintain an SBOM as part of every release — it's your security bill of lading."
  • SBOM — machine-readable inventory of all deps; formats: SPDX & CycloneDX; instantly reveals vulnerable libraries
  • Integrate FindSecurityBugs or SonarQube in CI — catch vulnerabilities at the PR stage, not post-deployment
  • Track EOL at endoflife.date — treat framework end-of-life with same urgency as a Critical CVE
  • VEX files suppress false positives — declare CVEs not exploitable in your context; reduce alert fatigue
  • CVSS score ≠ real-world risk — context (exposure, exploitability) determines actual priority

Day 02 · Session 01 ★ Favourite
Speaker: A N M Bazlur Rahman

Virtual Threads vs Reactive

"Virtual threads make blocking-style code scale — Reactive-level throughput with imperative readability."
  • Virtual Threads (Java 21+) — JVM-managed, ~few KB each; write blocking code naturally, millions can coexist
  • Choose VT for I/O-bound workloads (REST APIs, DB-heavy services) — it's now the right default
  • Keep Reactive only when you need built-in backpressure — streaming pipelines, slow-consumer signalling
  • Replace synchronized with ReentrantLock — prevents carrier thread pinning in VT-heavy code
  • CPU-bound work doesn't benefit from either — use ForkJoinPool or parallel streams instead

Day 02 · Session 02 ★ Favourite
Speaker: Benoît des Ligneris

GitOps with FluxCD & Kubeseal

"GitOps shifts the question: not 'what did someone deploy?' but 'what does Git say should be deployed?'"
  • Git = single source of truth for cluster state — every change is a commit with full history and rollback
  • Pull-based deployment — FluxCD watches the repo; CI never holds cluster credentials, reducing attack surface
  • Kubeseal — asymmetric encryption makes secrets safe to commit; only the cluster can decrypt them
  • Reconciliation self-heals drift — manual kubectl changes get reverted automatically
  • Disaster recovery for free — point a new cluster at the repo and reconciliation rebuilds everything

Day 03 · Session 01 ★ Favourite
Speaker: Cedric Clyburn

Automating with MCP Servers

"MCP turns your infrastructure into a conversational interface — ask questions, the AI runs the right commands."
  • Model Context Protocol — open standard giving AI typed access to tools (kubectl, GitHub, Prometheus, Postgres, Slack)
  • Chain servers for cross-tool workflows in one prompt: "find memory-hungry pods → look up GitHub configs → open Jira ticket"
  • Local stdio servers run with your own credentials — zero infrastructure, zero extra auth, just a process on your machine
  • Compounding effect — each server added multiplies AI's cross-stack reasoning; vendor-neutral and growing fast

Day 03 · Session 05 ★ Favourite
Speaker: Carl Lapierre

The Art of Context Engineering

output = LLM(context) — No hidden state, no memory. Every quality problem is a context problem.
  • Structure context with XML delimiters — models parse delimited sections more reliably than prose instructions
  • Critical instructions at start AND end — the middle of long contexts receives less model attention
  • Compress and summarise history — context rot is real; accumulated irrelevant turns silently degrade performance
  • Split complex tasks into focused sub-calls — map-reduce patterns beat one giant prompt every time
  • Retrieve, don't inject — don't include memory by default; retrieve only what's relevant to the current task

Client & Server
Implementation
